Tipos de bases de datos

Existen principalmente tres tipos de bases de datos, aunque son dos las que se suelen utilizar más

Qué son los tipos de datos relacionales

Son las más comunes y se basan en el modelo relacional. Organizan los datos en tablas estructuradas con filas y columnas interrelacionadas. Utilizan claves primarias y foráneas para establecer relaciones entre las tablas. Ejemplos populares de sistemas de gestión de bases de datos relacionales son MySQL, Oracle, PostgreSQL, MariaDB, SQLServer.

Se caracterizan por ser una coleción ordenada de registros que se van a organizar en un conjunto de tablas. Una tabla es el elemento principal de una base de datos relacional, es muy parecida a una hoja de cálcula. Los registros de información se organizan en filas y columnas. Las tablas es relacionan entre si, y para acceder a los datos utilizamos el lenguaje de consulta estructurada (SQL, Structure Query Language).

Con SQL vamos a poder obtener y alterar los datos de una forma organizada, teniendo en cuenta la estructura de la base de datos a la cual se está trabajando.

Estas tablas también se organizan a través de identificadores, es decir, cada tabla, elige a un campo como identificador único, y es el que estable la relacion con las otras tablas de la base de datos.

Qué son los tipos de datos no relacionales

Son bases de datos que no siguen el modelo relacional y proporcionan una forma más flexible de almacenar y recuperar datos. No requieren un esquema fijo y permiten manejar grandes volúmenes de datos no estructurados o semi-estructurados. Algunos tipos de bases de datos NoSQL son las de tipo clave-valor, de documentos, de columnas amplias y de grafos. Ejemplos de sistemas NoSQL incluyen MongoDB, Cassandra y Redis.

Se caracterizan por estar diseñadas para modelar datos con estructuras más específicas y no necesitan que los datos se relacionen los unos con los otros. Cada unidad funciona de forma independiente, y son más sencillas a la hora de acceder y ordenar.

Al no tener una estructura definida y exacta como las base de datos relacionales, se puede tener redundancia de datos (datos repetidos). Esto lo permiten porque lo que buscan estas bases de datos es mejorar el rendimiento y se prioriza el acceso por sobre la integridad de los datos. Intenta mostrar la mayor cantidad de datos en el menor tiempo posible.

El formato más popular que se utiliza en este tipo de bases de datos es el de un documento. Un documento es un objeto organizado en claves-valores para acceder a la información de manera sencilla.

A las bases de datos no relacionales se les asocia el concpeto de NoSQL, pero en realidad no quiere decir NO SQL, sino Not Only SQL, es decir, no existe solamente bases de datos en SQL.

Qué son los tipos de datos orientados a objetos

No se suelen utilizar tanto. Están diseñadas para almacenar y manipular objetos complejos, como imágenes, audio y video. Estas bases de datos permiten guardar los objetos directamente en la base de datos sin necesidad de descomponerlos en datos más simples. Ejemplos de bases de datos orientadas a objetos son db4o y Versant.
