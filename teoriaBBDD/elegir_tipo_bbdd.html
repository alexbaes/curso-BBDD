Como elegir un tipo de BBDD u otro

Si se quiere desarrollar una aplicación de tipo contable, inventario de productos, información de clientes, se adaptará mejor una base de datos de tipo relacional. Habrá más de una tabla que tenga relación con el resto.

Si se quiere desarrollar una aplicación que se necesite almacenar los datos, no necesiten estar relacionados y no tengamos la certeza de la estructura, debemos elegir una base de datos de tipo no relacional.

Ejemplo, una base de datos para almacenar las estadísticas de comportamiento de un usuario al visitar un sitio, o los datos biométricos. La información puede variar de una persona a otra, dependiendo de sus dinámicas con el sitio, de su salud.

Otro ejemplo, la creación de una galería de fotos en Instagram. Cada usuario puede subir la cantidad de fotos que quiera (respetando el límite de 10), pero uno puede subir 5, otro 2 y otro 10, no están obligados a subir 10 foto s.

Otro ejemplo, las estadísticas del progreso de una partida de videojuego. La información se almacena pero no se relaciona entre sí.

Resumiendo

Se aconseja escoger SQL

  • Mejor opción cuando necesitas mantener relaciones complejas entre los datos.
  • Ideal para aplicaciones con una estructura de datos definida y constante.
  • Útil cuando la consistencia y la integridad de los datos son críticas.
  • Adecuado para aplicaciones que requieren consultas complejas y transacciones ACID (Atomicidad, Consistencia, Aislamiento, Durabilidad).
  • Bueno para garantizar la coherencia de los datos en entornos de alta concurrencia.

Se aconseja escoger NoSQL

  • Recomendado cuando tienes una gran cantidad de datos no estructurados o semiestructurados.
  • Útil en casos donde la escalabilidad y el rendimiento son prioritarios.
  • Adecuado para aplicaciones que requieren alta velocidad de lectura/escritura y flexibilidad en el esquema de datos.
  • Ideal para entornos de desarrollo ágiles donde los requisitos de datos pueden cambiar rápidamente.
  • Bueno para almacenar datos de tipo clave-valor, documentos, columnas amplias o grafos.
